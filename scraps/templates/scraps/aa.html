Git

git init = 명령어로 로컬 저장소를 생성

.git = 버전 관리에 필요한 요소가 다 들어있음

  • 홈 디렉토리에서 ‘git init’을 실행하지 않는다 (desktop에서 하지 말라는뜻, 무조건 폴더 만들어서 거기서 해라)
  • Git 저장소 폴더 내의 폴더에서 ‘git init’을 하지 않는다. (중복 git 안되도록)
  • rm -rf .git = git 다 날려버림

README.md 생성하기

  • 새 폴더를 만들고 readme.md 파일 생성
  • 커밋 한다
  1. Working Directory - 내가 작업하고 있는 실제 디렉토리
  1. Staging Area - 커밋으로 남기고 싶은 특정 버전으로 관리하고 싶은 파일이 있는 곳
  1. Repository - 커밋들이 저장되는 곳
  • working directory에 있는 문서 → git add → staging Area에 들어감 → git commit → repository에 들어감

  • git status : git 상황을 알려줌
  • git add README.md : git에 파일을 올리는 거임
  • git add . : 수정된 모든 파일을 staging에 올림
  • git commit -m '쓰고싶은 말’ : 커밋을 할때는 반드시 말을 써야됨 (-m 은 message말하는거)

  • commit하기전 반드시 해야하는 두가지
  1. git config --global user.name ‘쓰고싶은 이름'
  1. git config --global user.email ‘쓰고싶은 이메일’
  1. git config --global -l : 등록한 이름과 이메일이 나옴

  • git log : 지금까지한 로그 볼 수 있음 - HEAD = 최신의 commit
  • git log --oneline : 로그 한줄보기
  • git restore README.md : 수정전으로 돌아가기

GitHub에 git 올리기

  1. git 로컬 파일 만들고 커밋하기
  1. git remote add origin https://github.com/godzz733/TIL.git (여기서 origin은 관례적으로 쓰는 이름임)
  1. git remote -v 로 잘 됫는지 확인
  1. git push origin master 하면 깃헙으로 커밋됨
  1. git push -u origin master 한번하면 그다음부턴 git push만 해도 됨
  1. 싸피에서 git clone github 주소 치면 클론됨
  1. 집에서는 git clone https://github.com/godzz733/git_home.git . 하면 폴더 안만들고 클론됨
  1. 집에서 push로 업데이트
  1. 다시 강의실 오면 git pull origin master 로 pull해오기

git에 커밋안할 것들(용량이 크거나 개인정보 등)

  1. git 폴더에 touch .gitignore 로 .gitignore라는 파일을 만듬
  1. 숨기고 싶은 파일을 .gitignore에 쓰면됨 (단 add를 하기전에 해야됨)
  1. ex) a.txt(a.txt파일을 커밋 안함), folder/ (폴더를 커밋안함), *.txt(txt 확장자를 다 커밋안함)
  1. 또는 인터넷에 gitignore.io 들어가서 해도됨
